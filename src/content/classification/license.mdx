# 라이선스별 분류

라이선스는 OS의 *수용성·생태계·정부 채택*을 결정한다.

## 오픈소스

| 라이선스 | 특징 | 대표 OS |
|---|---|---|
| **GPL 계열** | 파생 작품도 같은 라이선스로 공개해야 (copyleft) | Linux 커널, 대부분 GNU 도구 |
| **BSD/MIT 계열** | 사용·수정·재배포 자유, 상용 클로즈드도 가능 | FreeBSD, OpenBSD, NetBSD, Darwin |
| **Apache 2.0** | BSD-like + 특허 명시 | Android(AOSP 일부), Zephyr |
| **이중 라이선스** | 상황별 두 라이선스 중 선택 | Qt, MySQL |

## 상용 (proprietary)

소스 비공개. 사용 계약 + 라이선스 비용.

- Microsoft Windows
- macOS / iOS (소스 일부 Darwin은 오픈, GUI는 비공개)
- IBM z/OS
- Oracle Solaris (Solaris 11 이후 클로즈드)
- VxWorks, QNX (상용 RTOS)

## 정부·공공 OS — 한국 사례

Windows 종속 탈피를 목표로 정부가 적극 후원한 한국 OS 생태계.

### 구름 플랫폼 (Gooroom)
- **개발**: 한국전자통신연구원(ETRI) 산하 국가보안기술연구소, 2015~
- **기반**: Debian Linux
- **용도**: 클라우드 업무 환경 단말. 경찰·검찰·국방부 등 *보안 요구 기관*
- **특징**: 안전한 클라우드 업무 환경 접속용. 오픈소스

### TmaxOS (티맥스OS)
- **개발**: TmaxOS사 (티맥스 데이터 인수)
- **용도**: Windows 종속 탈피 목표. 자체 워드프로세서·웹브라우저 포함
- **파생**: Tmax Gooroom — 티맥스OS가 구름 플랫폼 기반 변형

### 하모니카 (HamoniKR)
- **개발**: 미래창조과학부 지원, 인베슘
- **기반**: Linux Mint (Debian/Ubuntu 계열)
- **용도**: 한글화·한국 환경 특화. *일반 공공기관* 대상
- **공공기관 도입 정책**: 보안 강조 기관(군·국방부 등)은 구름, 일반 공공기관은 하모니카로 분리

## 라이선스가 결정하는 것

- **서버 시장 Linux 51%+** — GPL의 *비용 0 + 자유 수정* 덕분.
- **슈퍼컴 100% Linux** — 연구기관이 *커널 패치* 가능해야 함.
- **임베디드 우위 BSD/MIT** — Sony PS5(FreeBSD)·Nintendo Switch — *상용 제품에 자유 재배포*.
- **정부 OS** — 국가 안보·종속 탈피 의도로 Linux 기반 채택.
