# 개요 — 분류의 다섯 축

운영체제(Operating System, OS)는 **하드웨어와 응용 프로그램 사이의 중간 계층**으로,
자원(CPU/메모리/저장장치/입출력)을 관리하고 응용에 표준 인터페이스를 제공한다.
같은 OS라도 보는 관점에 따라 여러 가지로 분류된다.

본 자료실은 다음 다섯 개 축으로 분류한다.

| 축 | 기준 | 분류 예시 |
|---|---|---|
| **① 시대별** | 등장 시기 + 기술 패러다임 | 1세대(직접 입력) → 5세대(모바일/클라우드/AI) |
| **② 용도별** | 대상 환경/시스템 | 데스크탑 · 서버 · 모바일 · 임베디드 · 실시간 · 메인프레임 · 슈퍼컴 |
| **③ 응용 분야별** | 산업/기기 | 자동차 · IoT · 게임 콘솔 · 의료 · 항공우주 |
| **④ 구조별** | 커널 설계 방식 | 모놀리식 · 마이크로 · 하이브리드 · 엑소 · 나노/유니 |
| **⑤ 라이선스별** | 오픈소스 vs 상용 | 오픈(Linux/BSD) vs 상용(Windows/macOS) |

## 왜 여러 축이 필요한가

한 축만 가지고는 *전체 풍경*을 그릴 수 없다. 예를 들어:

- **시대별**로만 보면 Linux와 Windows 11이 같은 5세대 OS지만, *용도*는 매우 다르다.
- **용도별**로 보면 서버용 RHEL과 임베디드 Yocto Linux가 다른 영역이지만, *구조*는 같은
  모놀리식 Linux 커널이다.
- **응용 분야별**로 보면 QNX(자동차)와 VxWorks(항공) 모두 실시간 OS지만, *라이선스*는
  둘 다 상용이라 오픈 RTOS(FreeRTOS, Zephyr)와 구분된다.

각 절에서 한 축을 깊게 보고, 마지막 [요약](/classification/summary)에서 대표 OS
12개를 다섯 축 전체에 한눈에 매핑한다.
